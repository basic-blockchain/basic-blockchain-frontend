## 1. Goal & non-goals

### Goal

Sixth screen-phase of v2. Migrates the four blockchain-primitive
views (Chain, Mempool, Nodes, Health) to consume the Phase 7.1
atoms, AND consolidates `StatusBadge.vue` into `BaseBadge` —
closing the deprecation flagged in 7.1.b §10.

### In scope

- **Migrate 4 views**:
  - `src/views/ChainView.vue` (390 LOC)
  - `src/views/MempoolView.vue` (311 LOC)
  - `src/views/NodesView.vue` (336 LOC)
  - `src/views/HealthView.vue` (176 LOC)
- **Consolidate StatusBadge → BaseBadge**: HealthView is the only
  component-level consumer (verified via strict
  `import.*StatusBadge` grep). The migration maps the v1 status
  enum to BaseBadge tones:
  - `ok` → `success`
  - `degraded` → `warning`
  - `error` → `danger`
  - `n/a` → `neutral`
- **Delete `src/components/atoms/StatusBadge.vue`** after
  HealthView migration confirms zero remaining consumers.
- **Delete `tests/unit/components/StatusBadge.test.ts`** in the
  same commit.
- Bigstat KPIs (where present) → `<BaseCard variant="bigstat">`.
- Inline tables → `<BaseTable>` with typed columns.
- Inline status pills / mempool state markers → `<BaseBadge>`.
- Action buttons (Refresh / Mine block / Resolve / etc.) →
  `<BaseButton>`.

### Non-goals

- **No migration of `ValidationView.vue`** (513 LOC). DESIGN-v2
  §5 #6 names only Chain/Mempool/Nodes; ValidationView is a
  sibling but heavier (handles a stepper-style validation
  workflow with sub-tabs). Defer to Phase 7.9 long-tail.
- **No backend / API changes** to `/chain`, `/mempool`,
  `/nodes`, `/health` endpoints.
- **No new composable extraction.** The 4 views have
  straightforward fetch + display patterns; nothing rises to
  composable-worthy complexity.
- **No `Pager` / `TabGroup` extraction.** Same trigger
  conditions as 7.4 / 7.5 — wait for additional consumers.
- **No `AssetPill` extraction** (the asset-pill recipe in
  MempoolView's currency column is the second consumer of the
  pattern after AdminWallets — promotion candidate for
  Phase 7.x, but not 7.7 to avoid scope creep).
- **No new dependency.**

- bigstat KPIs → `<BaseCard variant="bigstat">`
- Add-peer form button → `<BaseButton>`
- Peer connectivity status → `<BaseBadge>`
- **NodePanel kept** (already a Phase 6 organism; cosmetic
  refresh deferred)

### HealthView
- **StatusBadge consolidation** (the headline of the phase):
  ```diff
  - import StatusBadge from '@/components/atoms/StatusBadge.vue'
  + import BaseBadge from '@/components/atoms/BaseBadge.vue'

  - <StatusBadge :status="store.health.status" />
  + <BaseBadge :tone="healthTone(store.health.status)">
  +   {{ HEALTH_LABEL[store.health.status] }}
  + </BaseBadge>
  ```
- `healthTone()` helper maps ok→success, degraded→warning,
  error→danger, n/a→neutral.
- bigstat KPIs → `<BaseCard variant="bigstat">`
- Action buttons → `<BaseButton>`

## 4. Status tone mapping (BaseBadge soft + dot)

Same convention as 7.4 / 7.5:

| Health status | BaseBadge tone | Label |
|---|---|---|
| `ok` | `success` | "Operativo" |
| `degraded` | `warning` | "Degradado" |
| `error` | `danger` | "Error" |
| `n/a` | `neutral` | "N/D" |

| Mempool tx state | Tone | Label |
|---|---|---|
| `pending` | `warning` | "Pendiente" |
| `confirming` | `info` | "Confirmando" |
| `confirmed` | `success` | "Confirmado" |
| `failed` | `danger` | "Falló" |

| Node state | Tone | Label |
|---|---|---|
| `connected` | `success` | "Conectado" |
| `connecting` | `info` | "Conectando" |
| `disconnected` | `neutral` | "Desconectado" |
| `error` | `danger` | "Error" |
