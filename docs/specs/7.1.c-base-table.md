## 1. Goal & non-goals

### Goal

Add `src/components/atoms/BaseTable.vue` as the canonical v2 table
primitive: a generic, typed `<table>` with sticky header, hover row,
typed column defs, per-cell slots, a `row-actions` slot, an `empty`
state slot, and a `@sort` event hook. Pinned by DESIGN-v2 §4 row 5.

### In scope

- Greenfield `src/components/atoms/BaseTable.vue` (no existing
  BaseTable in tree per `grep -rn 'BaseTable' src/` — verified).
- Generic component (`<script setup lang="ts" generic="Row extends
  object">`) so columns and rows are type-checked against the
  consumer's data shape.
- Column metadata array (typed): `key`, `label`, `width?`, `align?`,
  `sortable?`, `num?`.
- Slots: `cell-<key>` (per-cell override), `row-actions` (trailing
  column), `empty` (no-data state).
- `@sort` event emitting `{ key, direction }`. Component does NOT
  sort internally — consumer owns ordering.
- Token-faithful styles mirroring prototype `.tbl` / `thead th` /
  `tbody td` / hover / selected / `.num` / `.row-actions`.
- Unit + snapshot tests covering column rendering, alignment,
  per-cell slots, row-actions, empty state, sort emission.

### Non-goals

- **No row selection / checkbox column.** The prototype's `.tbl-cb` +
  `.cbx` recipes and the associated `.bulk-bar` belong to a future
  7.1.x (`SelectableTable` wrapper or `BaseTable` selection extension)
  paired with the `BulkActionBar` primitive.
- **No pagination.** `.pager` in styles.css is its own sub-primitive
  and lands as a separate 7.1.x.
- **No internal sort logic.** `@sort` emits intent; the consumer
  reorders `rows` and passes them back. Keeps the component
  stateless and pure-render.
- **No `<div class="card">` / `.table-wrap` wrapper.** The component
  renders just the `<table>`. Wrapping is the consumer's call (some
  use `card`, some `table-wrap` with a toolbar header, some neither).
- **No migration of existing PrimeVue-based tables.** `MempoolTable`
  and `ConfirmedTransactionsTable` (organisms) keep using
  `primevue/datatable` until their screen-phase (Phase 7.7 —
  Chain / Mempool / Nodes views).
- **No migration of inline `<table class="tbl">` consumers** in
  `src/views/*`. They migrate per their screen-phase
  (Phase 7.4 / 7.5 / 7.6 / 7.7 / 7.9).

## 4. Column metadata

```ts
type Align = 'left' | 'center' | 'right'

interface Column<Row extends object> {
  key: keyof Row & string | string  // string fallback for synthetic columns
  label: string
  width?: string | number           // 'auto' | '120px' | 240 (px shorthand)
  align?: Align                     // default: 'left' (or 'right' if num)
  sortable?: boolean                // default: false — adds .col-sort affordance + click handler
  num?: boolean                     // default: false — right-align + tabular-nums + monospace
}
```

`key` doubles as:
- Property accessor on `Row` for default cell rendering
  (`row[key as keyof Row]`).
- Slot name for per-cell override: `<template #cell-amount="{ row,
  value }">…</template>`.
- Sort emission discriminator.

When `key` is a string not present on `Row` (synthetic column), the
default render is empty and the consumer is expected to provide the
`cell-<key>` slot.

## 5. Props / Slots / Events

### Props (TypeScript)

```ts
type SortDirection = 'asc' | 'desc'

interface BaseTableProps<Row extends object> {
  columns: Column<Row>[]
  rows: Row[]
  rowKey?: keyof Row & string | ((row: Row) => string | number)  // default: index
  rowClass?: (row: Row, index: number) => string | string[] | undefined
  sortKey?: string                  // controlled — currently sorted column key
  sortDirection?: SortDirection     // controlled — current sort direction
  showHead?: boolean                // default: true — set false for "naked" body-only tables
                                    // (e.g., UserWalletDetail movements list in the prototype)
}
```

`rowKey` defaults to the row index, which is acceptable for static
tables but will warn in dev when `rows` is reordered (consumers
SHOULD pass a stable accessor for animated/reordered tables).

### Slots

| Slot | Scope | Purpose |
|---|---|---|
| `cell-<column.key>` | `{ row: Row, value: unknown, index: number, column: Column<Row> }` | Per-cell custom render. If omitted, the default render is `String(row[key])`. |
| `row-actions` | `{ row: Row, index: number }` | Trailing column rendered after all `columns` entries. Rendered inside a `<td class="base-tbl__cell--actions">`. Header cell for this column is empty. Slot omitted = no actions column at all (the column does not render). |
| `empty` | — | Rendered as a single full-width `<tr><td colspan="N">` when `rows.length === 0`. Default content: muted "No data" text. |

### Events

```ts
defineEmits<{
  (e: 'sort', payload: { key: string; direction: SortDirection }): void
  (e: 'row-click', payload: { row: Row; index: number; event: MouseEvent }): void
}>()
```

Sort behaviour:
- Click on a sortable header emits `sort` with the new direction:
  - First click on an unsorted column → `'asc'`.
  - Click on the currently-asc-sorted column → `'desc'`.
  - Click on the currently-desc-sorted column → reverts to `'asc'`
    (no "unset" state in 7.1.c — defer if a consumer asks for it).
- The component does NOT sort `rows`. The consumer reorders and
  passes `rows` + `sortKey` + `sortDirection` back.

`row-click` fires on `<tr>` click. Consumers wire navigation, drawer
opening, etc. Default behaviour: no-op. Clicks on cells that contain
interactive elements (buttons, links) bubble normally; consumers can
`event.stopPropagation()` on those elements to avoid double activation.

## 6. Behavior

- Pure render. Sort direction is controlled via props, not internal
  state.
- `<thead>` element is omitted entirely when `showHead=false` (not
  rendered as empty — keeps the DOM minimal for naked tables).
- Column `width` accepts string CSS (`'120px'`, `'30%'`) or a number
  treated as pixels. Applied via `<colgroup><col>` so the
  consumer-provided widths win over the browser's default
  auto-layout heuristic.
- The `is-selected` row class is opt-in via `rowClass`; the
  component does not own selection.

## 7. Accessibility

- Native `<table>` semantics; no ARIA grid overlay. Keep it simple
  unless a real use-case for `role="grid"` surfaces in 7.4+.
- Sortable headers render as `<th>` with a nested `<button
  class="base-tbl__sort-btn">`. The button is the keyboard target;
  `aria-sort` on the `<th>` reflects the controlled state
  (`'none' | 'ascending' | 'descending'`).
- `<tr>` click handler does NOT make rows focusable by default
  (rows are not interactive elements per ARIA). If a consumer wires
  `row-click` for navigation, they SHOULD include a real link or
  button inside the row so keyboard users have a target — calling
  out in the consumer's PR, not enforced by BaseTable.
- Empty state slot is inside a `<tr><td>` with `role="row"` /
  `role="cell"` implicit from the native elements; screen readers
  announce it as a normal table row.

## 8. Test plan

### Unit tests (`BaseTable.spec.ts`)

- renders one `<tr>` per row
- renders columns in the declared order
- defaults to `String(row[key])` when no `cell-<key>` slot is provided
- uses the `cell-<key>` slot when provided, with the correct scope
  (`row`, `value`, `index`, `column`)
- applies `.base-tbl__cell--num` when `column.num=true`
- applies `column.align` to the cell text-align
- renders `<colgroup>` widths when `column.width` is provided
- omits `<thead>` when `showHead=false`
- renders the `row-actions` trailing column ONLY when the slot is
  provided
- renders the `empty` slot when `rows.length === 0`; default empty
  text appears when the slot is omitted
- emits `sort` with `'asc'` on first click of a sortable header
- emits `sort` with `'desc'` when clicking the asc-sorted header
- emits `sort` with `'asc'` when clicking the desc-sorted header
- does NOT emit `sort` for non-sortable headers
- sets `aria-sort` on the sortable header per controlled state
- emits `row-click` with `{ row, index, event }` on `<tr>` click
- applies `rowClass` output to the `<tr>` class list

### Snapshot test (`BaseTable.snap.spec.ts`)

Six snapshots (smaller matrix than 7.1.a/b — tables have larger DOM
output and snapshot diffs are noisier):

1. Plain table — 3 columns, 3 rows, no slots.
2. With `row-actions` slot — adds the trailing column.
3. With per-cell slot — `cell-amount` overrides default.
4. With `column.num` + `column.width` — numeric column, fixed widths.
5. With `showHead=false` — body-only.
6. Empty state — `rows=[]`, no `empty` slot (default text).

### Manual validation (PR description checklist)

- Light + dark theme: header background, border colors, hover
  background render per tokens.
- Sticky header: scroll a tall table inside a 200px-tall wrapper —
  header stays pinned.
- Keyboard nav: Tab into sortable header `<button>`, Enter triggers
  sort; row-click does NOT capture Tab focus on rows.
- Long-content rows: very long cell text wraps inside the cell
  without breaking sticky-header layout.
