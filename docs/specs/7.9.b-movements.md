## 1. Goal & non-goals

### Goal

Second long-tail sub-phase of v2. Migrate the movement views
(AdminMovementsView and AdminSendsView) to consume Phase 7.1 atoms
without changing data sources or behavior.

### In scope

- `src/views/AdminMovementsView.vue`
- `src/views/AdminSendsView.vue`

Shared migration rules (both views):
- Bigstat KPIs -> `<BaseCard variant="bigstat">`
- Inline tables -> `<BaseTable>` + `cell-<key>` slots
- Status pills -> `<BaseBadge>` (tone mapping per view)
- Inline buttons -> `<BaseButton>`
- Drop scoped CSS for `.bigstat*`, `.data-table*`, `.status-badge*`,
  `.btn*`, `.panel*` once absorbed by atoms.

### Non-goals

- No backend/API changes.
- No router changes.
- No new dependency.
- No Pager extraction (no paging UI in these views).
- No AssetPill extraction (no new consumer here).

## 4. Props / structure changes

### 4.1 AdminMovementsView -> BaseTable columns

```ts
const columns = [
  { key: 'type', label: 'Tipo' },
  { key: 'sender', label: 'De' },
  { key: 'receiver', label: 'Para' },
  { key: 'amount', label: 'Monto', num: true },
  { key: 'usd', label: 'USD', num: true },
  { key: 'status', label: 'Estado' },
  { key: 'when', label: 'Bloque / Cuando' },
]
```

Slot map:
- `cell-type`: keep the icon + label layout (`mv-type`) inline.
- `cell-sender` / `cell-receiver`: truncated mono addresses.
- `cell-amount`: native amount + currency tag.
- `cell-usd`: `fmtUsd` or em-dash.
- `cell-status`: `BaseBadge` with tone mapping:
  - `completed` -> `success`
  - `pending` -> `warning`
- `cell-when`: block index or timestamp fallback.

### 4.2 AdminSendsView -> BaseTable columns

```ts
const columns = [
  { key: 'type', label: 'Tipo' },
  { key: 'from', label: 'De' },
  { key: 'to', label: 'Para' },
  { key: 'amount', label: 'Monto', num: true },
  { key: 'ref', label: 'Hash / Ref' },
  { key: 'status', label: 'Estado' },
  { key: 'when', label: 'Cuando' },
]
```

Slot map:
- `cell-type`: `BaseBadge` tone mapping:
  - `Interno` -> `neutral`
  - `On-chain` -> `info`
  - `TesorerĂ­a` -> `warning`
- `cell-status`: `BaseBadge` tone mapping:
  - `completed` -> `success`
  - `pending` -> `warning`
  - `failed` -> `danger`
- `cell-from` / `cell-to`: truncated mono addresses.
- `cell-amount`: numeric formatting (unchanged).
- `cell-ref`: mono reference.
- `cell-when`: timestamp (unchanged).
