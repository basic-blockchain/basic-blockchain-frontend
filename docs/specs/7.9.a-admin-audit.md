## 1. Goal and non-goals

### Goal

First long-tail screen of Phase 7.9. Migrate
`src/views/AdminAuditView.vue` to the Phase 7.1 base components
without changing behavior or data flow.

### In scope

- KPI row (4 items) -> `<BaseCard variant="bigstat">`.
- Toolbar container -> `<BaseCard variant="default">` or
  `variant="ghost"` if already available in BaseCard.
- Table shell -> `<BaseCard variant="default" padding="none">` and
  `<BaseTable>` with typed columns.
- Action buttons (Actualizar) -> `<BaseButton>`.
- Category pill in the "Actor" column -> `<BaseBadge>` with tone
  derived from `getCat()`.
- Maintain current filters (search + category) and counts.

### Non-goals

- No API or store changes to `listAuditLog`.
- No column reordering or data shape changes.
- No new dependencies.
- No extraction of toolbar/search components.

## 2. Reference

### (A) Current implementation

- `AdminAuditView.vue` (table + KPI row + toolbar; pure view logic).

### (B) Design

DESIGN-v2 does not change audit layout; the migration is chrome swap

## 3. Migration map

### KPI row

- `.bigstat-row` -> `<BaseCard variant="bigstat">` x4.

### Toolbar

- Replace `.flow-card.toolbar` wrapper with `<BaseCard>`.
- Keep search input and category select unchanged (only class cleanup
  if the base card already provides padding).

### Table

- Replace `.panel` + `.tbl` with `<BaseCard padding="none">` and
  `<BaseTable>`.
- Define columns array for: `created_at`, `actor_id`, `action`, `origin`.
- Keep `formatDetail()` and `getCat()` logic as-is.
- Map the category badge to `BaseBadge`:
  - tone per category: `accent`, `info`, `warning`, `danger`, `success`,
    or `neutral` fallback.
