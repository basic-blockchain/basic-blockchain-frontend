## 1. Goal & non-goals

### Goal

Consolidate the existing `src/components/atoms/BaseButton.vue` into the v2
contract declared in DESIGN-v2 §4 row 1 (`variant: primary | secondary |
danger | ghost`, `size: sm | md`, `loading`). The component is the canonical
button primitive consumed by every Phase 7.2+ screen migration.

### In scope

- Reconcile the existing `BaseButton.vue` API (`size: default | sm | icon`,
  no `loading`) with the DESIGN-v2 contract.
- Retokenize hardcoded `0.12s` transitions to `var(--duration-fast)` +
  `var(--ease-out)` from the Phase 7.0 token deltas.
- Add slot anatomy for leading/trailing icons.
- Add `loading` state backed by the existing `.spinner` class from
  `propuesta_refactorizacion/styles.css`.
- Migrate the single existing call-site (`src/components/molecules/
  SeedPhraseModal.vue`) in the **implementation** PR (7.1.a impl), not in
  this spec PR.
- Ship at least one snapshot test + unit tests for `variant × size × state`
  rendering.

### Non-goals

- Migrating any other call-site. The 40 inline `.btn` usages across views
  stay on v1 styles until their screen-migration PR (7.2..7.9). This is
  enforced by §6 of DESIGN-v2 ("Migrate one screen or one base component
  per PR").
- Introducing a button group, split-button, dropdown-button, or toggle-button
  primitive — out of contract for 7.1.a. Tracked as 7.1 follow-ups (§10).
- Removing the legacy `.btn` / `.btn-primary` / `.btn-ghost` / `.btn-danger`
  classes from `src/assets/main.css`. They stay until the last v1 view migrates.
- Theming (light/dark) overrides beyond what the existing tokens already
  provide. The component renders via tokens; theme is the design-system's job.

## 3. Anatomy

```
┌─[ button.base-btn ]─────────────────────────────────────┐
│  ┌──────┐  ┌──────────────────┐  ┌──────┐  ┌──────┐    │
│  │ lead │  │ default slot     │  │ trail│  │ spin │    │
│  │ icon │  │ (label)          │  │ icon │  │ ner  │    │
│  │ slot │  │                  │  │ slot │  │ *    │    │
│  └──────┘  └──────────────────┘  └──────┘  └──────┘    │
└─────────────────────────────────────────────────────────┘
    ↑ gap: var(--space-sm) — 6px (matches prototype `.btn { gap: 6px }`)

* spinner replaces the leading-icon slot when `loading=true`.
  If no leading-icon slot is provided, spinner sits before the label.
```

Token map (host element):
- `padding`: `0 var(--space-md)` (10px) for `md`, `0 9px` for `sm` (no
  token for 9px yet — keep raw value, flagged as a follow-up token in §10).
- `gap`: `var(--space-sm)` (6px).
- `border-radius`: `var(--radius)` (6px).
- `border`: `1px solid var(--border)` (transparent for ghost).
- `box-shadow`: `var(--shadow-sm)` (none for ghost).
- `font`: `500 12.5px/1 var(--font-sans)` for `md`, `500 12px/1` for `sm`.
- `transition`: `background var(--duration-fast) var(--ease-out),
  border-color var(--duration-fast) var(--ease-out),
  color var(--duration-fast) var(--ease-out)`.
- Icons inside the button render at `14×14` (`md`) or `12×12` (`sm`) via a
  scoped `:deep(svg)` rule so consumers don't have to size their lucide
  imports.

## 4. Variants & states

`variant × size × state` matrix the implementation MUST cover. Tokens
referenced are resolved at runtime — light/dark fall out automatically.

### Variants (host backgrounds + borders)

| Variant | bg | color | border | shadow | Hover bg | Hover border |
|---|---|---|---|---|---|---|
| `primary`   | `var(--text)`      | `var(--bg)`    | `var(--text)` | `--shadow-sm` | `#2c2a26` | `#2c2a26` |
| `secondary` | `var(--surface)`   | `var(--text)`  | `var(--border)` | `--shadow-sm` | `var(--surface-2)` | `var(--border-strong)` |
| `danger`    | `var(--surface)`   | `var(--danger)`| `var(--border)` | `--shadow-sm` | `var(--danger-soft)` | `#f5c8c4` |
| `ghost`     | `transparent`      | `var(--text)`  | `transparent`   | none          | `var(--hover)` | `transparent` |

Notes:
- `primary` deliberately uses `var(--text)` (near-black) on light theme — it
  matches the prototype's "monochrome-on-paper" aesthetic. On dark theme,
  `var(--text)` resolves to a near-white, giving a dark-on-light inversion.
  This is **intentional** and confirmed by `styles.css:111` + the dark theme
  block in `design-system.css:124`.
- `#2c2a26` and `#f5c8c4` are raw hex values inherited from the prototype.
  They are not yet tokenized; introducing tokens for them is out of scope
  for 7.1.a (§10 follow-up).

### Sizes

| Size | Height | Padding-x | Font | Icon px |
|---|---|---|---|---|
| `md` (default) | 30px | 10px (`--space-md`) | 12.5px/1 | 14 |
| `sm`           | 26px | 9px (raw)           | 12px/1   | 12 |

`md` is the default. The "icon-only" form is **not** a size value — see
§5 `iconOnly` prop.

### States

| State | Trigger | Visual |
|---|---|---|
| `idle` | default | as per variant table |
| `hover` | `:hover:not(:disabled):not([data-loading])` | hover bg / border per variant |
| `focus-visible` | keyboard focus | `outline: 2px solid var(--accent); outline-offset: 2px` (unchanged from current impl) |
| `active` | `:active:not(:disabled)` | `transform: translateY(0.5px)` — INFERIDA, see §10 |
| `disabled` | `:disabled` | `opacity: 0.45; cursor: not-allowed; pointer-events: none for hover` |
| `loading` | `loading=true` (prop) | spinner replaces leading icon slot · pointer-events disabled · `aria-busy="true"` · text remains visible (does not collapse to spinner-only) |

`loading` implies `disabled` for click handling, but the host element keeps
`disabled` attribute set ONLY when the consumer passes `disabled` — so
screen readers announce "busy" (loading) vs. "dimmed/unavailable" (disabled)
distinctly. See §7.

## 5. Props / Slots / Events

### Props (TypeScript)

```ts
interface BaseButtonProps {
  variant?: 'primary' | 'secondary' | 'danger' | 'ghost'  // default: 'secondary'
  size?: 'sm' | 'md'                                      // default: 'md'
  loading?: boolean                                       // default: false
  disabled?: boolean                                      // default: false
  iconOnly?: boolean                                      // default: false — square aspect, hides label visually; consumer MUST pass `aria-label` via $attrs
  block?: boolean                                         // default: false — width:100%, justify-content:center
  type?: 'button' | 'submit' | 'reset'                    // default: 'button'
}
```

### Slots

| Slot | Purpose | Notes |
|---|---|---|
| (default) | Label text or arbitrary inline content | When `iconOnly=true` the slot is rendered inside `.sr-only` for screen-readers; consumers SHOULD still place readable text here. |
| `leading` | Icon before the label | Replaced by spinner when `loading=true`. |
| `trailing` | Icon after the label | Not replaced by spinner. |

### Events

The host `<button>` element is rendered directly — no event wrapping. All
native button events (`click`, `mousedown`, `keydown`, etc.) reach the
consumer via `v-on` listener inheritance (Vue 3 fallthrough attrs).

`click` is suppressed (preventDefault + no emit) while `loading=true` OR
`disabled=true`. This is enforced in template via `:disabled` binding, not
by a JS guard, so keyboard activation is also blocked.

## 6. Behavior

### Loading

- Spinner reuses the existing `.spinner` class from `styles.css` (lines for
  `.spinner` + `@keyframes sp`). The class will be promoted into a shared
  utility or inlined into `BaseButton.vue` scoped styles — the
  implementation PR decides; both are spec-compliant.
- `loading` flips `aria-busy="true"` on the host.
- The button MUST NOT change width when entering `loading` if a leading
  slot was provided — the spinner replaces the slot in-place. When no
  leading slot is provided, width MAY grow by `12px + 6px` (spinner + gap).
- Click handlers attached by the consumer DO NOT fire while loading.

### Disabled

- Native `:disabled` attribute.
- Hover / active styles do not apply (`:not(:disabled)` selectors).
- Cursor: `not-allowed`. Pointer-events: `none` is **not** set — leaving
  pointer-events on lets browser tooltips on the title attribute work.

### Focus & keyboard

- Default browser button keyboard behavior preserved (`Space`/`Enter`).
- `focus-visible` outline only — no outline on mouse focus.
- `Tab` order: native, no `tabindex` override.

### Hover

- All transitions use `var(--duration-fast)` (120ms) + `var(--ease-out)`.
- Hover styles do not apply while `loading` or `disabled`.

## 7. Accessibility

- Role: native `<button>`. No `role="button"` on non-button elements.
- `type` defaults to `"button"` to avoid accidental form submission — same
  as the current implementation.
- `loading` ⇒ `aria-busy="true"` on host.
- `iconOnly=true` ⇒ the default slot content is wrapped in `.sr-only` so
  the accessible name comes from the slot. If the consumer passes
  `aria-label` it wins (fallthrough). The implementation MUST `console.warn`
  in dev when `iconOnly=true` and neither slot text nor `aria-label` is
  present — a docs-style guard, not a runtime throw.
- Contrast: every variant pair listed in §4 has been picked from the v1
  token palette which already meets WCAG AA 4.5:1 (verified in DESIGN.md
  Phase 5e). Spec amendment required if a future token shift breaks this.
- Focus ring: `outline: 2px solid var(--accent)` with `outline-offset: 2px`
  passes WCAG 2.4.7 against every variant background.

## 8. Test plan

### Unit tests (`BaseButton.spec.ts`)

Each row is one `it()`:

- renders the default slot
- applies `variant` class for each of `primary | secondary | danger | ghost`
- applies `size` class for each of `sm | md`
- applies `block` class when `block=true`
- applies `iconOnly` class when `iconOnly=true`
- emits native `click` when clicked
- does NOT emit `click` when `disabled=true`
- does NOT emit `click` when `loading=true`
- sets `aria-busy="true"` when `loading=true`
- renders the `leading` slot in idle state
- replaces the `leading` slot with a spinner when `loading=true`
- keeps the `trailing` slot visible when `loading=true`
- forwards `type` attribute (`button | submit | reset`)
- forwards arbitrary `aria-*` attributes via fallthrough

### Snapshot test (`BaseButton.snap.spec.ts`)

One snapshot per cell of `variant × size × { idle | loading | disabled }`
= 4 × 2 × 3 = 24 snapshots. Snapshot the rendered HTML, not styles.

### Manual validation (PR description checklist)

- Light theme: every variant renders with expected contrast.
- Dark theme: every variant renders with expected contrast.
- Tab focus reaches the button; focus-visible ring shows on keyboard, not
  on mouse.
- `loading` spinner spins continuously (no animation pause on re-render).
- `iconOnly` button announces its `aria-label` via screen reader (tested
  with NVDA on Windows or VoiceOver on macOS).
- SeedPhraseModal (the only existing consumer) renders unchanged
  visually after migration.

Playwright smoke is **not** introduced in 7.1.a — the stack decision per
DESIGN-v2 §6 ("Phase 7.1.a will decide") is deferred to 7.1.b unless the
reviewer requests otherwise.

## 10. Out-of-scope follow-ups

Items observed while writing this spec that belong in a different sub-phase:

### Spec amendments (small, may land before the impl PR)

- **`lg` size or block-only?** The prototype auth screens use inline
  `height: 36/38` + `width: 100%` for Login / Register CTAs. This spec
  formalizes the second axis (`block`) but does NOT introduce `lg`. If
  during 7.2 (Login migration) the visual diff against the prototype is
  unacceptable at 30px, amend this spec to add `size: 'lg'` (height 36px)
  and bump the matrix. INFERIDA — confirm at 7.2 review.
- **`:active` transform.** The prototype CSS does not declare an `:active`
  state. This spec proposes `translateY(0.5px)`. INFERIDA — reviewer can
  drop it without amending the spec; it's a polish detail.

### 7.1 follow-up components

- `ButtonGroup` (split actions, segmented control) — used by the toolbar
  tabs in `styles.css`'s `.tabs` block. Likely 7.1.x or absorbed into 7.4
  (AdminUsersView) when the bulk-action footer ships.
- `IconButton` ergonomic wrapper — currently `iconOnly` covers it.
  Reconsider only if the boilerplate (`<BaseButton iconOnly aria-label="…">`)
  becomes noisy in 7.2+.

### Token follow-ups

- 9px horizontal padding for `size="sm"` has no spacing token. Either
  introduce `--space-2xs: 2px` to compose `9px = 4 + 5` (awkward) or accept
  the raw value. Lean: raw, until a second `sm` consumer asks for the same.
- `#2c2a26` (primary hover) and `#f5c8c4` (danger hover border) are raw
  hexes inherited from the prototype. Tokenize when a second consumer needs
  them (Phase 7.4 / `BaseTable` row-hover proposals).

### Legacy cleanup

- The 40+ inline `.btn` consumers in `src/views/*` and
  `src/components/flows/*` migrate to `<BaseButton>` as part of their
  respective screen migration PRs (7.2..7.9), NOT in 7.1.a. The legacy
  `.btn` CSS rules in `src/assets/main.css` are deleted in the PR that
  migrates the LAST consumer, per DESIGN-v2 §6 ("No dead code at merge").
