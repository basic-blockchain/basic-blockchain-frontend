## 1. Goal & non-goals

### Goal

Migrar `src/views/ValidationView.vue` al sistema de componentes base (v2)
para cerrar el ultimo view pendiente del long tail. Mantener la logica
actual (validacion de cadena, bloque, nodo y transaccion) sin cambios.

### In scope

- KPI row a `<BaseCard variant="bigstat">`.
- Botones primarios/ghost a `<BaseButton>`.
- Paneles y secciones a `<BaseCard variant="default" padding="none">`.
- Lista de historial dentro de un card con header consistente (`panel-h`).
- Remover CSS legacy para `.panel`, `.bigstat`, `.btn*`, `.tbl` si aplica.

### Non-goals

- No cambios en stores, endpoints o reglas de validacion.
- No cambios en `InputNumber` / `InputText` (PrimeVue) ni en su
  comportamiento.
- No nuevos componentes ni dependencias.

---

## 2. Reference

### (A) Prototype

No hay prototipo especifico para `ValidationView`; se aplica la
convencion visual de las migraciones 7.9 (BaseCard + BaseButton).

### (B) Current implementation

- `ValidationView.vue` usa `.panel`, `.bigstat`, `.btn*` y estilos
  locales para cards y lista de historial.
- Validaciones: cadena (endpoint `/valid`), bloque, nodo, transaccion.

## 3. Migration map

### Header actions

- `Validar cadena` -> `<BaseButton variant="primary" size="sm">`
- `Exportar historial` -> `<BaseButton variant="ghost" size="sm">`

### KPI row

- 4 KPIs -> `<BaseCard variant="bigstat">` con `header` y `footer`.
- Mantener el color condicional para valido/invalido en el valor.

### Panels

- `Estado de la cadena` -> `<BaseCard variant="default" padding="none">`.
- `Validacion de bloque` -> `<BaseCard variant="default" padding="none">`.
- `Validacion de nodo` -> `<BaseCard variant="default" padding="none">`.
- `Validacion de transaccion` -> `<BaseCard variant="default" padding="none">`.
- `Historial de validaciones` -> `<BaseCard variant="default" padding="none">`.
