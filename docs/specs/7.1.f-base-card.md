## 1. Goal & non-goals

### Goal

Reshape `src/components/atoms/BaseCard.vue` from a flat one-slot
wrapper to a slot-composed container with header / default / footer
slots and a `bigstat` variant per DESIGN-v2 §4 row 4. This is the
sixth and final base-component sub-phase of Phase 7.1; after merge,
Phase 7.2 (Login migration) can start consuming the full atom set.

### In scope

- Rewrite `BaseCard.vue` props to `{ variant?: 'default' | 'bigstat',
  padding?: 'default' | 'none' | 'sm' }`. `variant` is new;
  `padding` survives from the existing API.
- Add three slots: `header`, default (body), `footer`.
- The `bigstat` variant restyles padding and inner typography (label
  / value / sub) but uses the same slot anatomy. Consumers compose
  freely — no enforced `<div class="lb">` markup.
- Retokenize hardcoded `14px 16px` / `8px 12px` paddings to use
  Phase 7.0 spacing tokens where possible (raw values kept inside
  the `bigstat` variant if they don't map cleanly).
- Unit + snapshot tests covering variant × padding × slot
  composition.

### Non-goals

- **No migration of inline `<div class="card">` / `.panel` /
  `.bigstat` consumers.** 24 view-level files carry these CSS
  classes; each migrates per its screen-phase (7.2 / 7.3 / 7.4 /
  7.5 / 7.7 / 7.9), per DESIGN-v2 §6.
- **No `StatCard` / `BigStatCard` separate primitive.** DESIGN-v2
  §4 explicitly says "bigstat becomes a card variant". If the
  composition turns out to be too loose during screen migrations,
  add a thin `StatCard` wrapper (label/value/sub as props) in a
  follow-up — not in 7.1.f.
- **No `panel` variant.** The prototype `.panel` recipe is
  visually equivalent to the `.card` recipe; consolidating both
  into the default variant is enough. If a Phase 7.x screen
  surfaces a real distinction (different border, different
  shadow), revisit.
- **No clickable / interactive card.** Card stays a passive
  surface. If a consumer wants click-as-link semantics, wrap in
  `<router-link>` or `<a>` — not a BaseCard prop.

## 3. Anatomy

### Default variant

```
┌─[ div.base-card ]──────────────────────────────────────────────┐
│  ┌─[ slot:header ]───────────────────────────────────────────┐ │
│  │ optional — when omitted, the wrapper renders just body    │ │
│  └───────────────────────────────────────────────────────────┘ │
│  ┌─[ slot:default ]──────────────────────────────────────────┐ │
│  │ body                                                       │ │
│  └───────────────────────────────────────────────────────────┘ │
│  ┌─[ slot:footer ]───────────────────────────────────────────┐ │
│  │ optional — typically action row or summary                │ │
│  └───────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────────┘
```

### Bigstat variant

```
┌─[ div.base-card.base-card--bigstat ]───────────────────────────┐
│  ┌─[ slot:header  → styled as "label" (text-2 11.5px) ]──────┐ │
│  │ "En cola"                                                  │ │
│  └───────────────────────────────────────────────────────────┘ │
│  ┌─[ slot:default → styled as "value" (24px tabular-nums) ]──┐ │
│  │ 23                                                         │ │
│  └───────────────────────────────────────────────────────────┘ │
│  ┌─[ slot:footer  → styled as "sub" (text-3 11.5px) ]────────┐ │
│  │ 4 prioritarias                                             │ │
│  └───────────────────────────────────────────────────────────┘ │
```

Bigstat applies typography deltas to ONLY the wrapper element of
each slot (via `:deep()` or scoped descendant selectors). It does
NOT replace the slot content — consumers pass plain text and the
variant handles the styling.

Token map:

| Token | Default | Bigstat |
|---|---|---|
| Background | `var(--surface)` | `var(--surface)` |
| Border | `1px solid var(--border)` | `1px solid var(--border)` |
| Border radius | `var(--radius-lg)` | `var(--radius-lg)` |
| Shadow | `var(--shadow-sm)` | `var(--shadow-sm)` |
| Padding (`padding='default'`) | `14px 16px` (raw — preserved from existing) | `14px 16px 16px` (raw — prototype `.stat`) |
| Padding (`padding='sm'`) | `8px 12px` (raw — preserved) | n/a (bigstat ignores `sm`; see §6) |
| Padding (`padding='none'`) | `0` + `overflow: hidden` (preserved) | n/a |
| Header typography | inherits | `font: 500 11.5px/1.2 var(--font-sans); color: var(--text-2);` |
| Default typography | inherits | `font: 600 24px/1 var(--font-sans); font-variant-numeric: tabular-nums; letter-spacing: -0.02em; margin-top: 4px;` |
| Footer typography | inherits | `font: 500 11.5px/1.4 var(--font-sans); color: var(--text-3); margin-top: 4px;` |

The raw padding values (`14px 16px`, `8px 12px`) are inherited from
the existing v1 component and the prototype `.card` / `.bigstat`.
Tokenisation is a §10 follow-up — premature until a second card
recipe needs a different padding scale.

## 4. Props / Slots / Events

### Props (TypeScript)

```ts
type Variant = 'default' | 'bigstat'
type Padding = 'default' | 'none' | 'sm'

interface BaseCardProps {
  variant?: Variant     // default: 'default'
  padding?: Padding     // default: 'default'
}
```

### Slots

| Slot | Default | Bigstat | Notes |
|---|---|---|---|
| `header` | rendered above body (no styling) | "label" typography | Omitted → wrapper element not rendered. |
| (default) | body | "value" typography | Required by convention. |
| `footer` | rendered below body (no styling) | "sub" typography | Omitted → wrapper not rendered. |

### Events

None. BaseCard is a passive container.

## 5. Behavior

- Pure render. No state, no watchers, no emits.
- `variant='bigstat'` combined with `padding='sm'` or `'none'`
  silently uses the bigstat default padding. The variant wins
  because the bigstat typography assumes a specific vertical
  rhythm. A dev-mode `console.warn` flags the conflict to surface
  it early during screen migrations.
- Header / footer slots use `v-if="$slots.header"` /
  `v-if="$slots.footer"` so the wrapper element is omitted when
  the slot is unused — keeps the DOM minimal and avoids stray
  margin from empty wrappers.

---

## 6. Accessibility

- Root is `<div>`. No `role` injected — BaseCard is a generic
  container, and forcing `role="region"` would make every card
  show up as a landmark to AT users, which is noisy.
- Consumers that want landmark semantics pass `role="region"` +
  `aria-labelledby` via fallthrough attrs.
- The bigstat typography is purely visual — no AT-relevant
  semantics. If a consumer needs an AT-readable "label: value"
  pairing, they include both texts in the same slot so the reader
  announces them together.

## 7. Test plan

### Unit tests (`BaseCard.spec.ts`)

- renders the default slot as the body
- defaults to `variant='default'` and `padding='default'`
- applies `variant='bigstat'` class
- applies each of `padding='none'`, `'sm'`, `'default'` classes
- renders the `header` slot when provided; wrapper omitted when
  not provided
- renders the `footer` slot when provided; wrapper omitted when
  not provided
- logs a dev warning when `variant='bigstat'` is combined with a
  non-default `padding`
- forwards fallthrough attributes (`role`, `aria-labelledby`,
  `data-testid`)

### Snapshot tests (`BaseCard.snap.spec.ts`)

Six cells covering structural axes:

1. Default variant · default padding · body slot only.
2. Default variant · default padding · all three slots.
3. Default variant · `padding='sm'`.
4. Default variant · `padding='none'`.
5. Bigstat variant · all three slots (label / value / sub).
6. Bigstat variant · body-only (value only — no label, no sub).

### Manual validation (PR description checklist)

- Light + dark theme: default card chrome matches the prototype
  `.card` recipe at WCAG AA.
- Bigstat row: four BaseCards in a row render with matching
  vertical rhythm (label baseline aligned, value baseline aligned).
- Header / footer slots omitted: no stray margin or border at the
  top/bottom of the body.
