## 1. Goal & non-goals

### Goal

Greenfield two base primitives, paired in one PR per DESIGN-v2 §4
row 6:

- `src/components/atoms/BaseModal.vue` — the canonical overlay
  engine: Teleport to body, scrim, ESC, focus trap, body-scroll
  lock, slotted header / body / footer, `v-model:open`.
- `src/components/atoms/BaseDrawer.vue` — extends BaseModal with
  right-anchored slide-in panel + width control. Sets sane drawer
  defaults (anchor=right, width=720px) and overrides the scale-in
  animation for a horizontal slide.

DESIGN-v2 §5 #5 names BaseDrawer as the foundation of the Phase
7.6 drawer-unification (User → Wallet → Tx detail-panel
breadcrumb).

### In scope

- Greenfield `src/components/atoms/BaseModal.vue` and
  `src/components/atoms/BaseDrawer.vue`. Both verified absent via
  strict `import.*Base(Modal|Drawer)` grep.
- BaseModal handles: Teleport, scrim, dismiss (ESC + scrim click),
  focus trap with restore-on-close, body-scroll lock, slots
  `header` / default / `footer`, `v-model:open` two-way binding,
  optional `title` for `aria-label` fallback.
- BaseDrawer COMPOSES BaseModal — it renders BaseModal internally
  with a `variant="drawer"` mode that swaps the centred scale-in
  recipe for a right-anchored slide-in. The wrapper exposes
  `anchor: 'right' | 'left'` and a `width` prop (default 720px /
  max 96vw per the prototype).
- Unit + snapshot tests for both, covering open lifecycle,
  dismiss paths, focus management, and slot composition.

### Non-goals

- **No migration of existing modals / drawers.** The 23 in-tree
  surfaces using inline `Teleport`/`scrim`/`drawer` patterns
  (`UserDrawer`, `WalletDrawer`, `ProfileDrawer`,
  `ConfirmUserModal`, `SeedPhraseModal`, 11 flow components,
  `MiningNotification`, `CommandPalette`, and 2 view-inline
  overlays) stay on v1 chrome until their screen-phase. Each one
  carries domain content that's part of its screen migration —
  bundling them here would erase the screen-phase / base-phase
  separation that DESIGN-v2 §6 enforces.
- **No nested modals.** Stack depth = 1. If a flow needs nested
  confirmation (e.g., delete-from-inside-drawer), the outer
  surface should compose, not stack. Revisit if Phase 7.8 treasury
  flows force the issue.
- **No portal target other than `body`.** Teleport target is fixed.
- **No animation customisation prop.** The two animations
  (scale-in for modal, slide-in for drawer) are part of the
  variant, not overridable per-instance.
- **No `lazy` / `keep-alive` toggle.** BaseModal renders only when
  `open === true` (unmount on close). Consumers that need to
  preserve form state across close/reopen should hold that state
  in their own component, not inside the modal.

## 4. State & lifecycle

| Phase | Trigger | What happens |
|---|---|---|
| Closed → Opening | `open` flips `false → true` | `v-if` mounts the Teleport tree · body-scroll lock applied (`document.body.style.overflow = 'hidden'`) · previously-focused element saved · focus moves to first focusable element inside the modal (or the panel itself if none) · ESC listener registered on `window` |
| Opening → Open | CSS transition completes | no JS — purely visual |
| Open (Tab / Shift+Tab) | keydown | focus trap cycles within modal (`focus-trap` is hand-rolled — no library) |
| Open (ESC) | keydown | if `dismissable`, emits `update:open=false` + `close` |
| Open (scrim click) | mousedown on scrim element itself | if `dismissable`, same emits |
| Open → Closing | `open` flips `true → false` | ESC listener removed · focus restored to previously-focused element · body-scroll lock released · `v-if` unmounts after the transition ends |

Body-scroll lock counter: the implementation tracks a window-scoped
counter so nested-but-sequential modals don't release the lock
prematurely. Nested-simultaneous is non-goal (§1) but sequential
open/close patterns happen in flows like KYC review and treasury
approval.

## 5. Props / Slots / Events

### BaseModal

```ts
interface BaseModalProps {
  open: boolean                  // required, v-model target
  title?: string                 // a11y fallback (used in aria-label when header slot omitted)
  width?: string | number        // default '480px' (modal) | accepts '480px' | 480 (px shorthand)
  dismissable?: boolean          // default true — controls BOTH ESC and scrim click
}
```

### BaseDrawer (extends-by-composition)

```ts
interface BaseDrawerProps extends Omit<BaseModalProps, 'width'> {
  anchor?: 'right' | 'left'      // default 'right'
  width?: string | number        // default '720px'
}
```

### Slots (both)

| Slot | Purpose | Notes |
|---|---|---|
| `header` | Top section (title + close button + breadcrumbs) | If omitted: default header renders with `title` + a close button (the `×` glyph). |
| (default) | Body content | Scrolls vertically inside the panel when content overflows. |
| `footer` | Trailing section (typically action buttons) | Optional. No styling pressure when omitted. |

All slots receive no scope props — the component is a passive
shell.

### Events (both)

```ts
defineEmits<{
  'update:open': [value: boolean]   // v-model binding
  'close': []                        // semantic close — fires alongside update:open=false
  'opened': []                       // post-transition open hook (mirror of close timing)
}>()
```

`close` is emitted on EVERY dismiss path (ESC, scrim click,
consumer setting `open=false`). `update:open` matches.

## 6. Behavior

### Focus management (hand-rolled, no library)

- On open: capture `document.activeElement` as the restore target.
- Query focusable elements inside the panel via:
  `'a[href],button:not([disabled]),input:not([disabled]),select:not([disabled]),textarea:not([disabled]),[tabindex]:not([tabindex="-1"])'`.
- If at least one is found, focus the first.
- If none, focus the panel itself (`tabindex="-1"` on the panel).
- On Tab at the last focusable: wrap to first.
- On Shift+Tab at the first focusable: wrap to last.
- On close: focus the saved restore target if still in the DOM,
  else `document.body`.

### Dismiss handling

- Single `dismissable` prop covers ESC + scrim. Granular split
  (`closeOnEsc` / `closeOnScrim`) is a follow-up — every prototype
  consumer uses both or neither.
- Scrim click is filtered: the handler checks `event.target ===
  scrim element` (not `currentTarget`) so clicks on slot content
  that bubble up do not close.

### Body-scroll lock

- A module-scoped counter increments on open and decrements on
  close.
- When counter goes 0 → 1: save `document.body.style.overflow` and
  set to `'hidden'`.
- When counter goes 1 → 0: restore the saved value.

### Animation

- Vue `<Transition>` wraps the panel + scrim:
  - Modal: `enter-active` applies `scale-in 220ms ease-out`;
    `leave-active` applies `scale-out 120ms ease-in`.
  - Drawer: `enter-active` applies `translateX-in 220ms ease-out`;
    `leave-active` `translateX-out 120ms ease-in`.
- Scrim opacity follows the same `220ms / 120ms` symmetry.

## 7. Accessibility

- Root panel is `<div role="dialog" aria-modal="true">` (modal) or
  `<aside role="dialog" aria-modal="true">` (drawer).
- `aria-label` resolves in this order: explicit attribute on the
  component (fallthrough) > `title` prop > omitted (consumer is
  responsible). Component does NOT inject a default
  `aria-label="Modal"` because the prototype consumers always
  supply a meaningful label.
- Focus trap as described in §6.
- ESC closes (when `dismissable=true`). Documented in §6.
- Scrim is `aria-hidden="true"` to keep AT tree focused on the
  panel.
- Default close button (when `header` slot is omitted): renders
  with `aria-label="Cerrar"` (matches prototype Spanish strings).

## 8. Test plan

### Unit tests (`BaseModal.spec.ts`)

- mounts nothing when `open=false`
- mounts the Teleport tree when `open=true`
- renders the default slot (body)
- renders the `header` slot when provided; falls back to the
  built-in close button when omitted
- renders the `footer` slot when provided
- emits `update:open` and `close` on ESC keydown when `dismissable`
- emits `update:open` and `close` on scrim click when `dismissable`
- does NOT emit on scrim click when the target is a slot child
  (bubbled click)
- does NOT emit on ESC or scrim click when `dismissable=false`
- restores focus to the previously-focused element on close
- applies `document.body.style.overflow = 'hidden'` while open and
  restores on close
- handles sequential open/close pairs without leaking the
  body-scroll lock
- traps Tab focus within the panel (Tab at last → first; Shift+Tab
  at first → last)
- emits `opened` after the open transition runs

### Unit tests (`BaseDrawer.spec.ts`)

- defaults to `anchor='right'`
- renders with `anchor='left'` when set
- defaults to `width='720px'` and accepts a numeric override
- inherits dismiss behaviour from BaseModal (one happy-path ESC
  test is sufficient — the full matrix is covered in
  BaseModal.spec.ts)

### Snapshot tests

- `BaseModal.snap.spec.ts` — 4 cells: default (slots empty), with
  title prop, with header + footer slots, with custom width.
- `BaseDrawer.snap.spec.ts` — 3 cells: default (right, 720px),
  left anchor, custom width.

### Manual validation (PR description checklist)

- Light + dark theme: scrim opacity matches the prototype
  literal values; panel chrome reads at WCAG AA against scrim.
- Sequential open/close (modal → flow → modal again) does not
  leave the body scrolled-locked.
- Tab cycle stays inside the panel during long-form modal
  content (verify with `<input>` + multiple buttons in a slotted
  test fixture).
- ESC during a transition (mid-open animation) cleanly cancels;
  no orphan listeners.

## 10. Out-of-scope follow-ups

### 7.1 follow-up components / amendments

- **`--ease-in` token usage.** Phase 7.0 introduced `--ease-in`
  alongside `--ease-out`; both are available. The leave-animation
  uses `var(--ease-in)` per §9. If the resulting leave feels
  sluggish during PR review, tokenise the leave duration to
  `var(--duration-fast)` (120ms) — already proposed in §9.
- **Granular dismiss props** (`closeOnEsc` / `closeOnScrim`).
  Defer until a consumer asks for asymmetric dismiss.
- **Stacked modal support.** Genuine non-goal until a flow needs
  it. The body-scroll-lock counter is the only piece of state that
  needs revisiting; everything else is per-instance.
- **Restoring scroll position inside the panel** after a re-open.
  The `lazy unmount` rule (§1) means each open is a fresh tree —
  if a flow needs persistent scroll, hold a `scrollTop` ref in the
  consumer.

### Token follow-ups

- The raw `-16px 0 64px rgba(0,0,0,0.16)` drawer shadow and the
  two raw `rgba(20,18,12,0.{32,42})` scrim values are inherited
  from the prototype. Tokenise only when a second drawer-shaped
  primitive (sheet, side-popover) lands.
- Body-scroll-lock saved value lives in module scope (closure on
  the SFC). If multiple BaseModal instances run in the same
  bundle (Storybook, etc.) the counter still works because the
  module is shared.

### Legacy cleanup

- 23 in-tree surfaces (`UserDrawer.vue`, `WalletDrawer.vue`,
  `ProfileDrawer.vue`, `ConfirmUserModal.vue`,
  `SeedPhraseModal.vue`, 11 flow components,
  `MiningNotification.vue`, `CommandPalette.vue`, 2 view-inline
  overlays) migrate each in their respective screen-phase. The
  legacy `.modal-scrim` / `.modal` / `.drawer` / `.scrim` CSS in
  `src/assets/main.css` (and `propuesta_refactorizacion/styles.css`
  reference styles inlined into `App.vue`) stay until the last
  inline consumer migrates, per DESIGN-v2 §6.
