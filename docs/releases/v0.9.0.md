# v0.9.0 — Phase 7 Design System v2 & Atomic Migration

Pairs with simulator **v0.15.0**. This release closes the full Phase 7 frontend
effort: introduces Design v2 tokens, rewrites the base atom library, migrates
every view and flow to the new Atomic Design contract, wires the treasury
approval flow to the backend, and promotes the GitFlow chain through to `main`.

## Summary

Phase 7 was a cross-cutting rewrite of the UI layer. The legacy component
catalog and `.btn` styles were replaced with a unified set of atoms
(`BaseButton`, `BaseBadge`, `BaseTable`, `BaseModal`, `BaseDrawer`, `BaseCard`,
`Stepper`) built against Design v2 tokens, and every view (auth, admin, chain,
mempool, nodes, health, treasury, validation, etc.) plus drawers and modals
were migrated to that contract.

## Changes

### Design System v2 (Phase 7.0)

- New `DESIGN-v2.md` document formalizing the visual and API contract for the
  Phase 7 atoms.
- Design v2 tokens added: `surface-3`, `radius-pill`, the spacing scale and
  motion curves.

### Atom rewrites (Phase 7.1)

- **7.1.a `BaseButton`** — rewrite against the v2 contract with variants,
  sizes (including `lg`) and states; unit test suite + snapshot matrix.
- **7.1.b `BaseBadge`** — new `tone × variant` contract; `StatusBadge` flagged
  deprecated until its consolidation in 7.7.
- **7.1.c `BaseTable`** — greenfield atom with `rowKey` / `rowClass` / slots,
  unit tests and snapshot matrix.
- **7.1.d `Stepper`** — rewrite to the 7.1.d contract with consumer migration.
- **7.1.e `BaseModal` + `BaseDrawer`** — greenfield modal plus drawer wrapping
  it with a side variant; both with tests and snapshot matrix.
- **7.1.f `BaseCard`** — rewrite to the 7.1.f contract with header / body /
  footer slots.

### View and flow migration

- **7.2 Auth** — `LoginView`, `RegisterView` and `ActivateView` migrated to a
  new molecular `AuthLayout`, with `Stepper` and `BaseBadge`.
- **7.3 Admin dashboard** — `AdminView` migrated to 7.1 atoms with `Stepper`
  indicator; new `useDashboardFetchSteps` composable for tracking parallel
  fetches, with its own test suite.
- **7.4 AdminUsersView** — full migration to 7.1 atoms.
- **7.5 AdminWalletsView** — full migration to 7.1 atoms.
- **7.6 Drawers and modals** — `WalletDrawer`, `UserDrawer` and `ProfileDrawer`
  migrated to `BaseDrawer`; `ConfirmUserModal` migrated to `BaseModal`.
- **7.7 Chain / Mempool / Nodes / Health** — `ChainView`, `MempoolView`,
  `NodesView` and `HealthView` migrated to 7.1 atoms; `StatusBadge`
  consolidated into `BaseBadge`.
- **7.8 Treasury** — `AdminTreasuryView` migrated, Stepper error UX added,
  Mint + Treasury dual-sign flow aligned.
- **7.9 Admin closeout**:
  - 7.9.a Audit, compliance and KYC views migrated to
    `BaseCard` / `BaseButton` / `BaseTable`.
  - 7.9.b Movements and sends views migrated.
  - 7.9.c Admin configuration views migrated.
  - 7.9.d `ValidationView` migrated to `BaseCard` + `BaseButton` + `BaseTable`,
    with paginated history, CSV export and a safe timestamp formatter.

### Treasury approval flow

- `TreasuryApprovalFlow` wired to the backend (`initiate` / `approve` /
  `cancel`) and aligned with the simulator treasury state machine.
- Template parse fix (inline `@click`) and prior restore of the component to
  the `develop` baseline before cleanup.

### Cross-cutting UI cleanup

- Legacy `.btn` elements replaced by `BaseButton` across every flow
  (`ExchangeOrderFlow`, `TreasuryApprovalFlow`, other flows) and in
  `WalletDrawer`.
- `unify paginated list chrome` refactor — shared chrome for paginated lists
  built on `BaseTable`.

### Tooling, lint and tests

- Snapshot matrices added for every new atom (`BaseButton`, `BaseBadge`,
  `BaseTable`, `BaseModal`, `BaseDrawer`, `BaseCard`, `Stepper`, `AuthLayout`).
- Explicitly typed test helpers for `BaseModal` / `BaseDrawer`;
  `rowKey` / `rowClass` cast through VTU generic widening in `BaseTable`.
- ESLint autofix applied to `BaseButton`, `BaseModal`, `BaseTable`, drawers
  and migrated views.
- Stale `eslint-disable` directives removed from `BaseCard` and `Stepper`.

### Documentation

- `CLAUDE.md` now includes the **Local validation before pushing** section:
  `npm run build` must be the last step before `git push`, not an intermediate
  check.
- Phase 7 specs (`7.1.a`–`7.1.f`, `7.2`–`7.9.d`) added under `docs/specs/`.
- Roadmap updated: Phase 6 closed, Phase 7 opened with its sub-phase table and
  marked complete at the end.
- 7.9 closeout documented.

### Integration and promotion

- Upward GitFlow chain (`develop` → `qa` → `staging` → `production` → `main`)
  executed with branch protection contexts aligned to the real GitHub Actions
  job names (see `align develop required checks with real job names`).

## Notes

- This release closes Phase 7 on the frontend. All screen-migration debt to
  the new design system is paid down and merged to `main`.
- `StatusBadge` is consolidated into `BaseBadge` (removed after 7.7).
- Follow-up work can resume from the post-Phase-7 backlog without carrying
  any pending screen migration.
