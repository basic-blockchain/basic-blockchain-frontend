# v0.9.0 — Phase 7 Closeout & Treasury UX

Pairs with simulator **v0.15.0**. This release closes the Phase 7 UI batch,
integrates the treasury approval flow, and carries the GitFlow chain through to
`main`.

## Changes

### Integration & Promotion

- Completed the upward GitFlow chain (`develop` → `qa` → `staging` →
  `production` → `main`) with the branch protection contexts kept in sync with
  the active GitHub Actions workflows.
- Consolidated the Phase 7 screen migrations into the released `main` branch so
  the design-system closeout is now fully tagged.

### UI and Flow Updates

- Wired `TreasuryApprovalFlow` to the backend and aligned the UX with the
  simulator treasury state machine.
- Migrated `ValidationView` to the Phase 7 atoms, including the CSV export
  action and the safe timestamp formatter used in the table rows.
- Applied the final Phase 7.9 cleanup across the admin surface: audit,
  compliance, KYC, movements, sends, configuration, and validation views now
  share the same BaseCard / BaseButton / BaseTable structure.
- Kept the admin consistency polish from the Phase 6 tail: USD locale
  unification, severity-driven notifications, and the dashboard / movement
  formatting adjustments.

## Notes

- Release closes Phase 7 on the frontend.
- Next work can resume from the post-Phase-7 backlog without carrying any open
  screen migration debt.
