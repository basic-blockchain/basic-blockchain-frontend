# v0.7.0 — Auth + Wallet UI (Phase I.4)

Pairs with simulator **v0.13.0**. Adds full JWT authentication, BIP-39 wallet management, and signed-transfer UI.

## New features

### Authentication flow
- `LoginView` — username / password form; invalid credentials produce a toast, no redirect.
- `RegisterView` — creates a new account and navigates to the activation step.
- `ActivateView` — sets the initial password from the one-time activation code returned at registration.
- Authenticated users reaching `/login` or `/register` are redirected to `/wallet`.
- Unauthenticated users reaching any protected route are redirected to `/login`.

### JWT management
- `src/stores/auth.ts` (Pinia) — persists token + user profile in `localStorage` (`bb_auth`); survives page reload.
- `src/api/client.ts` interceptor — injects `Authorization: Bearer <token>` on every outbound request; clears storage and redirects to `/login` on HTTP 401.
- `GET /auth/me` is called after login to populate `display_name`, `roles`, and `user_id`.

### Wallet UI (`/wallet`)
- Lists all wallets owned by the current user with live balances and frozen status.
- **Create wallet** — calls `POST /wallets`; opens `SeedPhraseModal` that forces the user to acknowledge saving the 12-word recovery phrase before proceeding.
- **Signed transfer form** — takes sender wallet, receiver wallet ID, amount, nonce, and recovery phrase. The phrase is used locally to derive the secp256k1 keypair and sign the canonical transfer message; **the mnemonic is never sent to the server**. The hex signature is submitted via `POST /transactions/signed`.

### Admin UI (`/admin`, ADMIN role required)
- Lists all users with display name, role chips (ADMIN / OPERATOR / VIEWER — clickable to grant/revoke), and banned status.
- Ban / Unban button per user.
- **Mint tokens** form — calls `POST /admin/mint`; result enters the mempool and confirms on next mine.
- Non-ADMIN users navigating to `/admin` are redirected to `/wallet`.

### Sidebar updates
- Wallet link visible to authenticated users; Admin link visible to ADMIN only.
- User chip at the bottom of the sidebar shows display name, top role, and a sign-out button.
- Auth views (`/login`, `/register`, `/activate`) render full-screen without the sidebar.

## Dependencies added
- `@scure/bip39 ^2.2.0` — BIP-39 mnemonic validation and seed derivation.
- `@noble/curves ^2.2.0` — secp256k1 ECDSA signing (matches `coincurve` on the backend).
