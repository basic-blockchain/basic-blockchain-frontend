# v0.8.1 — Auth Token Persistence Fix

Pairs with simulator **v0.14.1**. Hotfix release addressing token persistence
edge case introduced in v0.8.0.

## Fixes

### Auth & Session Management

- Fixed JWT persistence during login flow to prevent spurious 401 errors when
  fetching user profile (`/auth/me`) immediately after token acquisition.
- Ensured token is available in session storage before any authenticated API calls.

## Validation

- All CI checks pass: lint, typecheck, test, build, SAST.
- Auth flow tested end-to-end with profile fetch on mount.

## Notes

- Recommendation: Upgrade from v0.8.0 to v0.8.1 to avoid login timing issues.
- Pairs with v0.14.1 backend release for consistency.
