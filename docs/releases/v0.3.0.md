# Release v0.3.0

## Summary

Accessibility hardening release — WCAG AA contrast across the dark theme, keyboard reachability for the chip atoms, ARIA semantics on the live status indicators, and a mobile navigation drawer so the dashboard stops being desktop-only. Notes written retroactively (2026-05-07).

## Included Changes

- **Mobile navigation.**
  - Hamburger drawer + overlay shell on small viewports.
  - Mobile-first responsive fixes for the Dashboard, Mempool view, and the metrics bar so they reflow rather than overflow.
- **ARIA semantics.**
  - WebSocket status indicator and the metrics bar gained `aria-live` regions so screen readers announce state changes.
  - `StatusBadge` and `MineButton` carry explicit `aria-*` attributes describing their state and busy/loading semantics.
  - App shell adds a "Skip to main" link plus full ARIA landmarks; new global `:focus-visible` ring style.
- **Keyboard reachability.**
  - `HashChip` is now keyboard-accessible — it can be focused and copied without a mouse.
- **Contrast / dark palette.**
  - `--text-muted` token upgraded to a colour that passes WCAG AA against the surface background.
  - `HashChip` light background replaced with dark-palette tokens, fixing the previous low-contrast affordance.
  - WCAG AA contrast audit table added to the UX requirements doc.
- **Tests.**
  - `makeUseWebSocketMock` callback signature corrected so the websocket composable test setup matches the production code.

## Validation

- Vitest suite green at the time of cut.
- Manual a11y audit covered keyboard tab order, screen-reader announcement of live regions, and the new contrast tokens against the WCAG AA threshold.

## Notes

- This is the foundation later phases (h1, h2) build on — no functional regressions, all changes are additive or contrast/semantics fixes.
