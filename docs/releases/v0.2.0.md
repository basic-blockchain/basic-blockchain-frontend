# Release v0.2.0

## Summary

First post-scaffold feature release. Adds a Validation Center with persistent history, a confirmed-transaction history list on the Mempool view, and makes WebSocket parse errors observable instead of silently swallowed. Notes written retroactively (2026-05-07) — at original tag time the GitHub Release was published with auto-generated PR bullets only.

## Included Changes

- **Validation Center.**
  - New `ValidationEvent` domain type and `useValidationHistoryStore` Pinia store, both unit-tested.
  - History timeline + export integrated into the Validation Center view, so operators can audit what was checked when.
- **Mempool — confirmed transaction history.**
  - New `ConfirmedTransaction` domain type and `useConfirmedTransactionsStore` (with unit tests) capture transactions as they leave the mempool.
  - New `ConfirmedTransactionsTable` organism renders the history alongside pending entries on the Mempool view.
  - The mining flow now persists confirmed transactions through to the store on every successful block.
- **WebSocket observability.**
  - `useBlockchainWebSocket` no longer silently catches malformed messages. Parse errors surface through a `lastError` ref so the UI (and tests) can react.
- **Docs.** Backend default port reference corrected from 8000 to 5000 in the user-facing docs.

## Validation

- Vitest suite green at the time of cut, including the two new store tests (`useValidationHistoryStore`, `useConfirmedTransactionsStore`).

## Notes

- The auto-flow that the v0.5.0 release later closed (rehydrating confirmed history from the backend on page reload) was *not* yet in place — at v0.2.0 the history table populated only from live WebSocket events. That gap is what v0.5.0 fixed.
